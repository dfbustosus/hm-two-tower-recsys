# Code of Conduct

## Our Pledge

We are committed to making this project a respectful, inclusive, and productive environment for everyone. This project involves machine learning, competition data, and security-sensitive workflows; collaboration must therefore be careful, evidence-driven, and professional.

## Expected Behavior

- Be respectful, direct, and constructive.
- Critique ideas, code, experiments, and assumptions; do not attack people.
- Prefer reproducible evidence over status, seniority, or model hype.
- Respect Kaggle data rules, privacy, licensing, and repository security boundaries.
- Disclose uncertainty clearly and ask for clarification rather than inventing facts.

## Unacceptable Behavior

- Harassment, insults, intimidation, or discriminatory language.
- Publishing private information, credentials, tokens, raw Kaggle data, or article images.
- Deliberately introducing temporal leakage, hidden data dependencies, or misleading metrics.
- Dismissing review feedback without addressing the underlying technical concern.
- Any conduct that makes the project unsafe or hostile for contributors.

## Enforcement

Maintainers may remove comments, reject contributions, restrict participation, or take other reasonable action when behavior violates this code of conduct.

To report a concern, contact the repository maintainers through GitHub. If the concern includes sensitive personal or security information, do not post details publicly; request a private maintainer contact or use GitHub private vulnerability reporting if it is enabled for the repository.

## Scope

This code applies in repository issues, pull requests, discussions, reviews, project communication channels, and any public context where someone represents the project.
